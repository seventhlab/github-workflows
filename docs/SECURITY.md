# Security Policy

## Supported Versions

Security fixes are applied to the active development line and latest supported release line.

| Version | Supported |
| --- | --- |
| `main` | :white_check_mark: |
| Latest major release line | :white_check_mark: |
| Older release lines | :x: |

## Reporting a Vulnerability

Do not open public issues for security reports.

Please email `security@seventhlab.com` with:

- vulnerability description
- reproduction steps or proof of concept
- potential impact
- suggested remediation (optional)

## Response Expectations

- Initial acknowledgement target: within 48 hours
- Triage and validation: as quickly as possible
- Fix and disclosure timeline: based on severity and impact

If validated, we will:

1. Develop and ship a fix.
2. Coordinate disclosure timing.
3. Credit the reporter (unless anonymity is requested).
